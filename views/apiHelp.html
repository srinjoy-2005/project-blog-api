Blog API Help

This page documents the available API endpoints for the blog application. The root route / serves this page, and the JSON API endpoints are mounted under /signup, /login, and /posts.

Authentication & User Management

POST/signup

Create a new user account.

Request body (JSON): 
{
  "username": "string",
  "fullname": "string",
  "password": "string"
}
Successful response: 201 Created 
{
  "id": 1,
  "username": "johndoe",
  "fullname": "John Doe",
  "accountCreationDate": "2026-06-13T..."
}
Errors:
  • 400 if validation fails.
  • 409 if the username already exists.

POST/login

Authenticate a user and receive a JWT token.

Request body (JSON): 
{
  "username": "string",
  "password": "string"
}
Successful response: 200 OK

The response includes an Authorization header with a bearer token and the authenticated user's profile.

{
  "user": {
    "id": 1,
    "username": "johndoe",
    "fullname": "John Doe",
    "accountCreationDate": "2026-06-13T..."
  }
}
Errors:
  • 400 if payload validation fails.
  • 401 if credentials are incorrect.

Posts

GET/posts

Retrieve all posts.

Behavior:
  • If no valid token is provided, only public fields are returned: id, title, and body.
  • If a valid token is provided, full post records are returned.
Response: 200 OK

POST/posts

Create a new post. Authentication is optional; unauthenticated posts are saved with author guest.

Request body (JSON): 
{
  "title": "string",
  "body": "string"
}
Response: 201 Created Errors:
  • 400 if title or body is missing.

GET/posts/:id

Retrieve a single post by numeric ID.

Response: 200 OK Errors:
  • 400 if the post ID is invalid.
  • 404 if the post is not found.

PUT/posts/:id

Update an existing post. Requires a valid JWT and the authenticated user must be the author.

Request body (JSON): 
{
  "title": "string",
  "body": "string"
}
Response: 200 OK Errors:
  • 400 if the post ID is invalid or required fields are missing.
  • 403 if the authenticated user is not the owner.
  • 404 if the post is not found.

DELETE/posts/:id

Delete a post by ID. Requires a valid JWT and ownership of the post.

Response: 204 No Content Errors:
  • 400 if the post ID is invalid.
  • 403 if the authenticated user is not the owner.
  • 404 if the post is not found.

Comments

All comment endpoints are nested under /posts/:id/comments.

GET/posts/:id/comments

List all comments for a specific post.

Response: 200 OK

GET/posts/:id/comments/:commentId

Retrieve a single comment by ID for a specified post.

Errors:
  • 404 if the comment is not found.

POST/posts/:id/comments

Add a comment to a post.

Request body (JSON): 
{
  "text": "string",
  "commenterUsername": "string"
}
Response: 201 Created Errors:
  • 400 if the post ID is invalid or required fields are missing.
  • 404 if the post or commenter username is not found.

DELETE/posts/:id/comments/:commentId

Delete a comment by ID from a specific post.

Response: 204 No Content Errors:
  • 404 if the comment is not found.

General Notes

  • Routes not found return 404 with JSON { "error": "Route not found" }.
  • Unhandled server errors return 500 with JSON { "error": "..." }.
